Forms overview

Are used to get user input.

Angular has 2 ways to implement forms.

  1. Template driven forms
  1. Reactive forms

Template driven forms

In template driven forms, the source of truth is the template.

Is quick to setup and easy to use and is best for small one time use forms.

It requires more configuration for unit testing.

[(ngModel)] is called banana box syntax. It allows to have 2 way data binding and is a combination of both @Input() and @Output() which means there should be a property named username and when the input form is changed, it will also change the value of the property.

Example | Template Driven Forms

app.component.ts 

@Component({
	imports: [FormsModule], 
	templateUrl: 'app.component.html'
})

export class AppComponent {
	username: ""; // initial value of 'username' used in the form
	password: ""; // initial value of 'password' used in the form 
}

app.component.html

<form name="loginForm">
  <label for="username">username: </label>
  <input type="text" id="" [(ngModel)]="username" />

  <label for="username">password: </label>
  <input type="password" id="" [(ngModel)]="password" />

  <button type="submit">Login</button>
</form>

Reactive Forms

Supports typing

Is reusable and also can share models (2 way data binding)

Allows more robust testing configuration

We need to define model in order to use reactive form using ReactiveFormsModule and FormGroup

example | Reactive Forms

app.component.ts

@Component({
	imports: [ReactiveFormsModule], 
	templateUrl: 'app.component.html'
})

export class AppComponent {
	// creating new FormGroup
	loginForm = new FormGroup({
		name: new FormControl(''), 
		email: new FormControl('')
	})

	handleSubmit(){
		console.log("handling form"); 
		// accessing the values of the loginForm FormGroup
		this.loginWithCredentials(this.loginForm.value)  
	}
}
Note: FormControl accepts only defined data type

app.component.html

<!-- implementing formGroup to form, adding submit event handler -->
<form name="loginForm" [formGroup]="loginForm" (ngSubmit)="handleSubmit()">
  <label for="username">Username: </label>
  <!-- reactive way to bind name -->
	<!-- not using property binding "[ ]" since the values are known are not passed dynamically -->
	<input type="text" formControlName="username" /> 
	

  <label for="password">Password: </label>
  <!-- reactive way to bind name -->
	<!-- not using property binding "[ ]" since the values are known are not passed dynamically -->
  <input type="text" formControlName="password" />

  <button type="submit">Login</button>
</form>

Assignment| Reactive Forms

solved assignment: 10-reactive-forms

app.component.ts

import { Component } from '@angular/core';
import { FormControl, FormGroup, ReactiveFormsModule } from '@angular/forms';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [ReactiveFormsModule],
  template: 'app.component.html',
  styles: [],
})
export class AppComponent {
  blogForm = new FormGroup({
    title: new FormControl(''),
    body: new FormControl(''),
  });

  handleFormSubmit() {
    const { body, title } = this.blogForm.value;
    this.postBlog(title, body);
  }

  postBlog(title: string | null | undefined, body: string | null | undefined) {
    console.log(`Posting blog titles ${title}, with the contents ${body}.`);
  }
}

app.component.html

    <article>
      <h1>Blog Post</h1>
      <form
        name="blogForm"
        [formGroup]="blogForm"
        (ngSubmit)="handleFormSubmit()"
      >
        <section>
          <label for="title">Post Title</label>
          <input type="text" id="title" formControlName="title" />

          <label for="body">Post Body</label>
          <textarea
            id="body"
            cols="30"
            rows="10"
            formControlName="body"
          ></textarea>
        </section>
        <button type="submit">Submit Post</button>
      </form>
    </article>

Dependency Injection (DI)

assignment covered in lecture: 11-dependency-injection

Is a design pattern mechanism in computer science and is not angular specific.

Used for creating and delivering some part of an app to other parts of an app that require them.

Is implemented with Injectable decorator.

syntax: 

import { Injectable } from "@angular/core"; 

@Injectable({
	providedIn: 'root', 
})

providedIn defines the scope where the part of code is available for. If it is root then it will be available in the whole application and is quite common to see. But it can also be limited within a component and its children or between components.
