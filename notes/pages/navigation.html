Routing

Angular has built-in, complete router available at @angular/router

Angular 17+ routing allows transition animation between routes too.

Route guard is available out of the box and can be configured in the route file itself.

Basic router configuration:

app.routes.ts

Defining /details route and rendering DetailsComponent for that route

import { Component } from '@angular/core'; 
import { Routes } from '@angular/router'; 
import { DetailsComponent} from './details/details.component'; 

export const routes: Routes = [
	{
		path: 'details', 
		component: DetailsComponent
	}
]

Router Outlet

router outlet is where the contents from the routes gets rendered.

It can be helpful when only a particular part of UI is getting changed while other components are still same like sidebar and navigation bar can be standard for all the components.

example:

src/app/app.component.html 

<app-header></app-header>
<div class="appBody">
  <app-sidebar></app-sidebar>
  <div>
    <router-outlet></router-outlet>
  </div>
</div>

RouterModule

The RouterModule has to be imported in the component file before being used. 

import { RouterModule, Routes } from '@angular/router';

@Component({
	imports: [RouterModule]
})

src/app.config.ts 

import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
	// provide the list of routes to be used in the app (dependency injection)
  providers: [provideRouter(routes)],
};

Dynamic Routes & Router Link

Router link

routerLink directive is used to add clickable links

Directives adds behavior to the component.

example:

using routerLink directive on a tag 

import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterModule } from '@angular/router';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [CommonModule, RouterModule],
  template: `
    <a routerLink="/details"></a>
    <router-outlet />
  `,
  styles: [],
})
export class AppComponent {
}

Dynamic Route

To use the dynamic route feature, withComponentInputBinding() has to be used in the router configuration. It allows to bind the route param to be used and set with @Input() which will then allow to set the value of param like a variable from the component file.

Example:

app.config.ts 

import { ApplicationConfig } from '@angular/core';
import { provideRouter, withComponentInputBinding } from '@angular/router';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [provideRouter(routes, withComponentInputBinding())],
};

example: getting value for :id in the path /details/:id dynamically with @Input() from DetailsComponent

routes.ts 

import { Component } from '@angular/core';
import { Routes } from '@angular/router';
import { DetailsComponent } from './details/details.component'; 

export const routes: Routes = [
	{
		path: 'details/:id', 
		component: DetailsComponent
	}
]

details/details.component.ts 

@Component({...})

export class DetailsComponent {
	productId = -1; // destination for route info

	// setting value of id in the router path 
	@Input()
		set id(value: number){
			this.productId = value; 
		}
}
