Signals

Angular updates the UI by itself without having to set some states like in React. It is done with zones. It is called zones based reactivity.

zones based reactivity had problem on large applications and on the scale, it was not easy to manage. That’s how signals were introduced.

Three reactive primitives:

  1. Signal
  1. Computed
  1. Effect

Signal

A value that can tell Angular when it changes.

Is capable of notifying it’s context of future changes in it’s value.

It allow to refresh UI of the only specific part.

example:

Note: while using signal in template file, parenthesis has to be used as signals act as functions
@Component({
	template: `<p> {{ lastnName() }}, {{ firstName() }} </p>`
})

export class Appcomponent {
	firstName = signal('Jessica'); 
	lastName = signal('Wesly'); 
}

Computed

Derive new value when one of the dependent signals change

Is good in terms of performance as it only recalculates if something changes

example: 
 @Component({
	template: `<p> {{ fullName() }} </p>`
})

export class Appcomponent {
	firstName = signal('Jessica'); 
	lastName = signal('Wesly'); 
	fullName = computed(()=> `${firstName} ${lastName}`);
}

Effect

An effect is a side-effectful operation which reads the value of zero or more signals

Is useful to debugging

It takes function as parameter

example:

logging the value of lastName each time it’s value is changed 

 @Component({
	template: `<p> {{ fullName() }} </p>`
})

export class Appcomponent {
	firstName = signal('Jessica'); 
	lastName = signal('Wesly'); 
	fullName = computed(()=> `${firstName} ${lastName}`); 
	effect(() => console.log('Updated' + lastName())); 
}

Deferrable Views

really nice documentation on deferrable views:

Two way data binding was one of the few things other frameworks adapted from Angular.

Lazy loading helps to keep initial data bundles sizes smaller. It makes sure that it doesn’t makes calls unless a trigger is passed which will initialize it.

example:

Load the <recommended-movies /> only when trigger is interacted

<button #loadMovie>Load Recommend Movies</button>

@defer (on interaction(loadMovie)) {
<h1>Recommended movie list</h1>
} @loading {
<p>Loading...</p>
}@error {
<p>Error...</p>
}@placeholder {
<p>Placeholder....</p>
}

Trigger can be implemented on various situations

  1. on idle
  1. on immediate
  1. on timer
  1. on viewport
  1. on interaction (used above)
  1. on hover

Is differ a promise?

Not really, it is implemented on template level. It will not load the template until provided condition is met while promise is something we will wait until it is resolved or rejected.
