Template Conditionals

Control flow with @if

It works just like if conditions in JavaScript.

example:

Basic usage with @if : 

<section> 
	@if(expression){
		<p> If case paragraph</p>
	}
	<p> Normal flow paragraph</p>
</section>

Using @if @else if and @else 

import { Component } from '@angular/core';
import { AccountInfo } from './account-info';

@Component({
  selector: 'app-root',
  standalone: true,
  template: `
    <article class="card">
      <h1 class="card-title">Frontend Masters Express</h1>
      <p class="card-number">0000 0000 0000 0000</p>
      <section class="membership-info">
        <p>
          <!-- name -->
          {{ account.name }}
        </p>
        <p>Valid Thru: {{ account.validThru }}</p>
        <p>CVV: {{ account.CVV }}</p>
        <p>
          <!-- membership status -->
          @if (account.membershipStatus === "gold") {
          <span class="badge gold">Gold</span>
          }@else if (account.membershipStatus==='platinum') {
          <span class="badge platinum">Platinum</span>
          }@else {
          <span class="badge silver">Silver</span>
          }
        </p>
      </section>
    </article>
  `,
  styleUrl: './app.component.css',
})
export class AppComponent {
  account: AccountInfo = {
    name: 'Melisa Evan',
    membershipStatus: 'gold',
    validThru: '12/2022',
    CVV: '123',
  };
}

Control flow with @switch

example: 

<!-- membership status -->
<p>
  @switch (account.membershipStatus) { 
		@case ('gold') {
	  <span span class="badge gold">Gold</span>
	  } 
		@case ('platinum') {
	  <span span class="badge platinum">Platinum</span>
	  } 
		@default () {
	  <span class="badge silver">Silver</span>
	  }
	}
</p>

Properties, Events & Output

Property binding

Properties in the templates can also be binded dynamically

example:

Setting property disabled to a button with [disabled] property binding and isDisabled value.

NOTE: the value is expression when used with property binding (i.e with [ and ] )
@Component(){
	template: `
		<button type="button" [disabled]="isDisabled">
			Submit 
		</button>
	`
}

export class AppComponent{
	isDisabled = false; 
}

Event binding

Event handling allows to responds to events in templates.

example:

handling click event on a button with (click) event binding and handleClick() handler. 

@Component(){
	template: `
		<button type="button" (click)="handleClick()">
			Submit 
		</button>
	`
}

export class AppComponent{
	handleClick(){
    console.log("click triggred")
  }  
}

Customizing Components with @Input()

@Input() allows to receive information to a component (like with props in React)

Is regular @Input() from previous versions of Angular. Nothing new with latest version.

documentation on @Input() : https://angular.io/guide/inputs-outputs#sending-data-to-a-child-component

Send Information from a child component with custom events (@Output())

@Output() allows to send data from a component through an event.

EventEmitter used in order to create an event with @Output()

example:

Implementing @Output() in child component and emitting event. 

@Component(){
	template: `
		<button type="button" (click)="addItem('Item 1')">
			Submit 
		</button>
	`
}

export class ChildComponent{
@Output() addItemEvent = new EventEmitter<string>(); 

	addItem(addedItem: string){
	  this.addItemEvent.emit(`Item added: ${addedItem}`)
	}
}

Consuming the event emitted on parent directory

NOTE: $event has to be the same where ever it is used. It holds the value received from event and passes to the respective function which is handling the event. 

@Component(){
	template: `
		<app-child (addItemEvent)="addItem($event)"/>
	`,
	imports: [ChildComponent]
}

export class ParentComponent{
	items: string[] = []

	addItem(addedItem: string){
	  this.items.push(addedItem); 
	}
}
